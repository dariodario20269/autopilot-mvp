# AutoPilot Production Deployment Guide

This guide provides comprehensive instructions for deploying the AutoPilot platform to a production environment.

---

## 1. Prerequisites

Before deploying to production, ensure you have:

- **Supabase Database**: PostgreSQL database configured with all tables migrated
- **Environment Variables**: All required API keys and credentials
- **Docker**: Docker and Docker Compose installed on your server
- **Domain**: A registered domain name (optional but recommended)
- **SSL Certificate**: HTTPS certificate (auto-generated by Let's Encrypt or provided by your hosting provider)

## 3. Docker Deployment

### Option A: Using Docker Compose (Recommended for Small to Medium Deployments)

1. **Copy files to your server**:
   ```bash
   scp -r /home/ubuntu/autopilot-rebuild/* user@your-server:/opt/autopilot/
   ```

2. **SSH into your server**:
   ```bash
   ssh user@your-server
   ```

3. **Navigate to the application directory**:
   ```bash
   cd /opt/autopilot
   ```

4. **Create `.env.production`** with all required variables (see section 2).

5. **Build and run with Docker Compose**:
   ```bash
   docker-compose up -d
   ```

6. **Verify the deployment**:
   ```bash
   docker-compose logs -f autopilot
   ```

### Option B: Using Kubernetes (Recommended for Large-Scale Deployments)

1. **Build Docker image**:
   ```bash
   docker build -t autopilot:latest .
   docker tag autopilot:latest your-registry/autopilot:latest
   docker push your-registry/autopilot:latest
   ```

2. **Create Kubernetes deployment manifest** (`k8s-deployment.yaml`):
   ```yaml
   apiVersion: apps/v1
   kind: Deployment
   metadata:
     name: autopilot
   spec:
     replicas: 3
     selector:
       matchLabels:
         app: autopilot
     template:
       metadata:
         labels:
           app: autopilot
       spec:
         containers:
         - name: autopilot
           image: your-registry/autopilot:latest
           ports:
           - containerPort: 3000
           env:
           - name: DATABASE_URL
             valueFrom:
               secretKeyRef:
                 name: autopilot-secrets
                 key: database-url
           - name: GEMINI_API_KEY
             valueFrom:
               secretKeyRef:
                 name: autopilot-secrets
                 key: gemini-api-key
           # ... add other environment variables
           livenessProbe:
             httpGet:
               path: /api/health
               port: 3000
             initialDelaySeconds: 30
             periodSeconds: 10
           readinessProbe:
             httpGet:
               path: /api/health
               port: 3000
             initialDelaySeconds: 5
             periodSeconds: 5
   ---
   apiVersion: v1
   kind: Service
   metadata:
     name: autopilot-service
   spec:
     selector:
       app: autopilot
     ports:
     - protocol: TCP
       port: 80
       targetPort: 3000
     type: LoadBalancer
   ```

3. **Deploy to Kubernetes**:
   ```bash
   kubectl apply -f k8s-deployment.yaml
   ```

## 4. Nginx Reverse Proxy Configuration

If using Nginx as a reverse proxy:

```nginx
upstream autopilot {
    server localhost:3000;
}

server {
    listen 80;
    server_name autopilot.yourdomain.com;
    
    # Redirect HTTP to HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name autopilot.yourdomain.com;
    
    # SSL certificates (use Let's Encrypt)
    ssl_certificate /etc/letsencrypt/live/autopilot.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/autopilot.yourdomain.com/privkey.pem;
    
    # Security headers
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-XSS-Protection "1; mode=block" always;
    
    # Proxy settings
    location / {
        proxy_pass http://autopilot;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 5. SSL/TLS Certificate Setup (Let's Encrypt)

1. **Install Certbot**:
   ```bash
   sudo apt-get install certbot python3-certbot-nginx
   ```

2. **Generate certificate**:
   ```bash
   sudo certbot certonly --nginx -d autopilot.yourdomain.com
   ```

3. **Auto-renewal**:
   ```bash
   sudo systemctl enable certbot.timer
   sudo systemctl start certbot.timer
   ```

## 6. Monitoring and Logging

### Using Docker Logs

```bash
# View logs
docker-compose logs -f autopilot

# View logs from specific time
docker-compose logs --since 2026-03-03T10:00:00 autopilot
```

### Using ELK Stack (Elasticsearch, Logstash, Kibana)

1. **Add logging configuration to docker-compose.yml**:
   ```yaml
   logging:
     driver: "json-file"
     options:
       max-size: "10m"
       max-file: "3"
   ```

2. **Set up centralized logging** with ELK or similar service.

### Health Checks

The application includes a health check endpoint at `/api/health`. Monitor this endpoint to ensure the application is running:

```bash
curl https://autopilot.yourdomain.com/api/health
```

## 7. Backup and Recovery

### Database Backups

1. **Automated Supabase backups**: Supabase automatically backs up your database daily.

2. **Manual backup**:
   ```bash
   pg_dump postgresql://postgres:password@host:port/database > backup.sql
   ```

3. **Restore from backup**:
   ```bash
   psql postgresql://postgres:password@host:port/database < backup.sql
   ```

### Application Backups

1. **Backup environment variables**:
   ```bash
   cp .env.production .env.production.backup
   ```

2. **Backup Docker volumes** (if using persistent storage):
   ```bash
   docker run --rm -v autopilot_data:/data -v $(pwd):/backup alpine tar czf /backup/autopilot_data.tar.gz -C /data .
   ```

## 8. Scaling and Performance Optimization

### Horizontal Scaling

1. **Load Balancing**: Use Nginx, HAProxy, or cloud provider load balancers.

2. **Multiple Instances**:
   ```bash
   docker-compose up -d --scale autopilot=3
   ```

### Performance Optimization

1. **Database Connection Pooling**: Use Supabase's built-in connection pooling.

2. **Caching**: Implement Redis caching for frequently accessed data.

3. **CDN**: Use CloudFlare or similar CDN for static assets.

## 9. Troubleshooting

### Application Won't Start

```bash
# Check logs
docker-compose logs autopilot

# Verify environment variables
docker-compose config

# Restart service
docker-compose restart autopilot
```

### Database Connection Issues

```bash
# Test connection
psql postgresql://postgres:password@host:port/database -c "SELECT 1"

# Check connection string in .env
cat .env.production | grep DATABASE_URL
```

### High Memory Usage

```bash
# Monitor container stats
docker stats autopilot

# Increase memory limit in docker-compose.yml
# Add: mem_limit: 2g
```

## 10. Security Best Practices

1. **Secrets Management**: Use environment variables or secrets management tools (e.g., HashiCorp Vault).

2. **API Rate Limiting**: Implement rate limiting on your API endpoints.

3. **CORS Configuration**: Restrict CORS to trusted domains only.

4. **SQL Injection Prevention**: Use parameterized queries (already implemented with Drizzle ORM).

5. **Regular Updates**: Keep dependencies and Docker images updated.

6. **Firewall Rules**: Restrict access to sensitive ports (e.g., database port).
